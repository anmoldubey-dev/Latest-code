TABLE 15: corrections
DATABASE: SQLite — LOCAL FILE (not Neon, not cloud)
FILE PATH: backend/speech/stt/feedback/corrections.db (or configured path)
PURPOSE: STT correction pairs to fix recurring speech-to-text misrecognitions
================================================================================

  COLUMN      TYPE     USED BY / DISPLAYS
  ----------- -------- -------------------------------------------------
  id          INTEGER  PK (AUTOINCREMENT)
  lang        TEXT     Language this correction applies to (e.g. "en", "hi")
  bad_text    TEXT     The wrong STT output pattern to find and replace
  corrected   TEXT     The correct replacement text
  hits        INTEGER  Count of how many times this correction has matched
                       (used to prioritise/audit high-frequency fixes)
  created_at  TEXT     Creation timestamp
  updated_at  TEXT     Last modification timestamp

  UNIQUE constraint on (lang, bad_text) — no duplicate correction pairs

FEATURES USING THIS TABLE:
  - STT feedback loop (feedback_loop.py) — after every speech transcription,
    all corrections for the detected language are loaded and applied as
    regex replacements to the raw STT output in real-time
  - Admin correction management — list_corrections() returns all pairs
    for display in admin correction console
  - add_correction() — agents/admins submit new correction pairs
