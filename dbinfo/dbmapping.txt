  U13. DATABASE TABLE REFERENCE — USER SPACE
================================================================================

TABLE: users
  Purpose  : User account (login, role, credits)
  Columns  : id, name, email, password, role, is_active, phone_number, created_at, credits
  Used by  : Login, Register, AuthContext (JWT validation)

TABLE: cc_sessions
  Purpose  : Every call record (browser, PSTN, direct, outbound)
  Columns  : id, session_id, room_id, caller_id (FK→cc_callers), department,
             status, call_duration, agent_id, created_at, ended_at,
             recording_url, recording_consent, wait_seconds, csat,
             queue_position, call_type
  Used by  : HistoryPanel, UserBrowserCall, RecordingController, DialerPanel,
             CallbackNotification
  Key statuses: queued, connected, ended, abandoned, transferred, completed

TABLE: cc_callers
  Purpose  : Caller / user identity (email → display_name)
  Columns  : id, email (UNIQUE), display_name, last_seen
  Used by  : HistoryPanel (caller_name), UserBrowserCall (initiate → UPSERT),
             CallbackNotification (token fetch)

TABLE: transcripts
  Purpose  : Per-utterance speech during call (ASR output)
  Columns  : id, call_id, session_id, speaker, text, sentiment, created_at
  Used by  : HistoryPanel (transcript modal + sentiment badge),
             CallerTranscriptSender (writes in real-time during call)

TABLE: call_summaries
  Purpose  : Post-call AI-generated summary and aggregated sentiment
  Columns  : session_id (UNIQUE), call_id, num_speakers, speakers (JSON),
             avg_sentiment, sentiment_label, total_turns, duration_secs
  Used by  : HistoryPanel (sentiment indirectly), generated on call end by backend

TABLE: outbound_queue
  Purpose  : Scheduled callback jobs (agent schedules → user gets notified)
  Columns  : id, user_email, department, status, assigned_agent, attempts,
             created_at, last_attempt, scheduler_email_sent
  Used by  : CallbackNotification (accept/reject), AppShell (direct-decline)

TABLE: admin_config
  Purpose  : Key-value system configuration
  Columns  : key, value
  Key rows : business_hours, holiday_active, holiday_message
  Used by  : UserBrowserCall (GET /api/cc/business-hours before initiating call)
